## Unit 0 Decisions

### Use a dedicated software-development-system repository
Decision:
Use one dedicated repository to store learning labs, project records, and future integration projects.

Reason:
A single organized repo makes progress visible and prevents random scattered learning.

### Use Git and GitHub from the beginning
Decision:
Track all learning work with Git and push to GitHub.

Reason:
Git builds professional habits early: commit history, restore safety, branches, remotes, and conflict control.

### Inspect before committing
Decision:
Run `git status`, `git diff`, and staged inspection commands before committing.

Reason:
This prevents accidental commits, wrong files, tool-generated files, and unclear project history.

### Use branches for risky or separate work
Decision:
Use branches when testing changes that should not immediately affect `master`.

Reason:
Branches protect the stable version and allow safe experimentation.
